게시글 등록(게시글 스키마): /api/groups/{groupId}/posts

1. 이름(name): string, required 필드
2. 제목(title): string, required 필드
3. 이미지(postImg, 한장) : string, 이미지 주소가 될 예정, required 필드
4. 본문(content) : string, required 필드
5. 태그(hashtag) : string, required 필드
6. 장소(place) : string, required 필드
7. 추억 공개 여부(public) : true/false, 디폴트는 true, required 필드
8. 비밀번호(password): string / 게시글의 경우에는 public에 상관 없이 반드시 입력되야 하는 required 필드임
9. createdAt(추억의 순간), updatedAt: 자동 생성
10. 추억 공감 수(likes) : number, 디폴트값 0, required 필드
11. 댓글 수(comments) : number, 디폴트값 0, required 필드

게시글(추억) 수정: PUT /api/posts/{postId}
- 비밀번호 입력 후, 등록 시 입력했던 비밀번호와 일치하면 수정 가능
- 리퀘스트를 받을 때, 입력 된 값을 넘겨받아 일치하면 수정(PUT) 리스폰스를 돌려주도록 하면 됨.

게시글(추억) 삭제: DELETE /api/posts/{postId}
- 비밀번호 입력 후, 등록 시 입력했던 비밀번호와 일치하면 삭제 가능
- 리퀘스트를 받을 때, 입력 된 값을 넘겨받아 일치하면 삭제(DELETE) 리스폰스를 돌려주도록 하면 될 것 같다.

게시글(추억) 목록 조회: GET /api/groups/{groupId}/posts
- 그룹 상세 조회를 할 경우 그 그룹에 해당되는 추억 목록이 같이 조회
- 각 추억의 닉네임, 추억 공개 여부, 제목, 이미지, 태그, 장소, 추억의 순간, 추억 공감수, 댓글수가 표시
- 조회와 검색은 프론트 관련 영역인 것으로 생각됨

게시글(추억) 상세 조회: GET /api/posts/{postId}
- 추억 목록에서 추억을 클릭할 경우 추억 상세 조회가 가능
- 비공개 게시글(추억)을 조회 시도 할 경우 비밀번호도 받아서 일치할 때만 성공 리스폰스를 돌려줌
- 닉네임, 제목, 이미지(한 장), 본문, 태그, 장소, 추억의 순간, 추억 공개 여부, 추억 공감수, 댓글수가 표시됨
- 공감 보내기 버튼을 클릭할 경우 그룹의 공감수를 높일 수 있으며, 공감은 클릭할 때마다 중복해서 보낼 수 있음
- 해당 추억의 댓글 목록이 조회됨
